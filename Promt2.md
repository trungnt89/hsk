BỘ NGUYÊN TẮC THI HÀNH SỬA ĐỔI MÃ NGUỒN (STRICT PROTOCOL):
--------------------------------------------
	1. Liệt kê thay đổi: Nêu rõ các dòng được sửa và lý do kỹ thuật.
	2. Kiểm tra Sai khác (Git Diff - Toàn bộ): * Lập bảng so sánh chi tiết cho tất cả các dòng từ dòng 1 đến hết (không được lược bỏ bằng dấu ...).
		Cấu trúc bảng: | Dòng STT | Nội dung Code Base (Cũ) | Nội dung Code Mới |.
		Báo cáo các thông số chính xác:
		Tổng số dòng trong Code Base: [Con số]
		Tổng số dòng đã thực hiện kiểm tra: [Con số]
		Kết luận: Số dòng kiểm tra là (Ít hơn / Nhiều hơn / Bằng) so với Code Base.
	3. Tự rà soát (Self-Correction): Nếu phát hiện bất kỳ sự sai khác nào ngoài nội dung đã liệt kê ở bước 2 (kể cả khoảng trắng hay comment), phải thực hiện sửa lại cho đến khi chỉ còn đúng những thay đổi được yêu cầu. Báo cáo: "Hoàn thành bước 3".
	4. Xuất mã nguồn (Full Output): Cung cấp toàn bộ mã nguồn sau khi sửa. Tuyệt đối không viết tắt, không lược bỏ bất kỳ đoạn nào.
	5. So sánh đối chiếu cuối cùng: So sánh trực tiếp mã nguồn ở bước 4 với Code Base một lần nữa. Liệt kê danh sách các điểm khác biệt duy nhất.
	6. Xác nhận vi phạm: Trả lời trực tiếp câu hỏi: "Có phát hiện hành vi tự ý sửa code ngoài yêu cầu hay không?".
